# 他人接入指南：Tune Engine + 单次循环 开箱即用

> 给**不是你本人**的同事/其他团队：如何在不写代码的情况下，通过 **Agent 引用** 或 **工作流引用** 接入。
>
> - **批量调优** → 本文第三节（Tune Engine）
> - **单次循环（在线重试）** → [单次循环-他人接入指南.md](单次循环-他人接入指南.md) ★ 同样通用可接入

---

## 一、你需要先有什么

| 条件 | 说明 |
|------|------|
| Tune Engine 已部署 | 由运维启动，你只填地址 |
| 平台能挂 MCP | 必须 |
| 平台能引用 Agent / 工作流 | 推荐 |
| 平台能挂 Skill | 推荐 |

**你不需要：** 写 Python、改 Tune Engine 源码、改平台底层。

## 二、运维：一次性部署（30 分钟）

### 方式 1：一键脚本

```bash
cd /path/to/project
bash scripts/start-all.sh
```

### 方式 2：Docker

```bash
docker compose up -d
```

### 验证

| 服务 | 地址 | 检查 |
|------|------|------|
| Tune Engine | `http://HOST:8100/docs` | 打开 API 文档 |
| check-mcp | `http://HOST:8200/health` | `{"status":"ok"}` |
| run-log-mcp | `http://HOST:8300/health` | `{"status":"ok"}` |

把 `HOST` 记下来，后面配置要用（内网 IP 或域名）。

## 三、业务方：三种接入方式（选一种）

### 方式 A：引用「调优助手 Agent」★ 最推荐

**适合：** 希望像用一个现成 Agent 一样，说一句话就能调优。

#### 步骤

1. 打开 `integrations/agents/tune-assistant.json`
2. 把里面所有 `YOUR_HOST` 换成实际地址，例如 `192.168.1.100`
3. 在平台 **新建 Agent**，名称：`调优助手`
4. 按 JSON 配置：
   - 挂 MCP：`http://HOST:8100/mcp`（tune-mcp）
   - 挂 MCP：`http://HOST:8300/mcp`（run-log-mcp，可选）
   - 挂 Skill：复制 `integrations/skills/auto-tune-operator/SKILL.md` 内容
   - 挂工作流：引用 `wf_tune_generic`（见方式 B 的工作流描述）
5. 在你的 **总 Agent** 里 **添加 Agent 引用** → 选择 `调优助手`

#### 使用

用户对总 Agent 说：

```text
帮我把合同解析调到 90% 准确率
```

总 Agent 规划 → 路由到 **调优助手** → 自动 `tune.start` → 返回最优版本。

```text
┌──────────┐    引用     ┌──────────────┐    MCP     ┌─────────────┐
│  总 Agent │ ─────────> │  调优助手 Agent │ ────────> │ Tune Engine │
└──────────┘            └──────────────┘            └─────────────┘
```

---

### 方式 B：引用「通用调优工作流」

**适合：** 平台习惯用工作流编排，希望调优是一个标准工作流节点。

#### 步骤

1. 打开 `integrations/workflows/wf_tune_generic.json`
2. 在平台 **新建工作流** `wf_tune_generic`，把 5 个 step 的 intent 复制进去
3. 总 Agent 挂 MCP：`http://HOST:8100/mcp`
4. 总 Agent 挂 Skill：`auto-tune-operator`
5. 总 Agent **引用工作流** `wf_tune_generic`，或规划描述里写：
   > 当用户说调优时，路由到 wf_tune_generic

#### 工作流 5 步（平台里照抄）

| 步 | 做什么 | 调什么 |
|----|--------|--------|
| 1 | 从 query 识别 target_id | 无 |
| 2 | 启动调优 | MCP `tune.start` |
| 3 | 查进度 | MCP `tune.status` |
| 4 | 拿结果 | MCP `tune.get_result` |
| 5 | 总结给用户 | 总结模型 |

---

### 方式 C：最轻量（只挂 MCP + 一句 Skill）

**适合：** 不想新建 Agent/工作流，只在现有总 Agent 上加能力。

1. 总 Agent 挂 MCP：`http://HOST:8100/mcp`
2. 在现有 Skill 里加一段：

```markdown
当用户说「调优」「优化到」「提升到」时：
1. 识别 target_id（合同解析=contract-parse）
2. 调用 MCP tune.start(target_id=..., async=true)
3. 轮询 tune.status
4. 调用 tune.get_result
5. 汇报结果
```

## 四、在线重试（单次循环）怎么给别人用

**同样是通用的，可开箱接入。** 详细文档：**[单次循环-他人接入指南.md](单次循环-他人接入指南.md)**

与 Tune Engine 对等的三种接法：

| 方式 | 配置文件 | 说明 |
|------|----------|------|
| 引用 Agent ★ | `integrations/agents/retry-assistant.json` | 在线重试助手 |
| 引用工作流 | `integrations/workflows/wf_retry_wrapper.json` | 一个壳包住所有业务 |
| 挂到业务 Agent | `skills/auto-retry-replan/SKILL.md` + 两个 MCP | 不改总 Agent |

依赖 MCP：`check-mcp :8200` + `run-log-mcp :8300`（`start-all.sh` 一并启动）

新业务：在 `integrations/task-registry.yaml` 加 `task_type` + `expected_fields`，**不用改 MCP 代码**。

```text
用户普通提问 → 总 Agent → 在线重试助手 / wf_retry_wrapper → 业务工作流
用户要调优   → 总 Agent → 调优助手 / wf_tune_generic
```

## 五、注册你自己的技能（让别人能调你的业务）

Tune Engine 要调你的「合同解析」，需要先在 Tune Engine 里**注册**（运维或你做一次）：

```bash
curl -X POST http://HOST:8100/api/v1/targets \
  -H "Content-Type: application/json" \
  -d @tune-engine/examples/contract-parse-target.json
```

把 `entry_ref` 改成你们平台真实的 Agent/工作流 ID。

以后别人说「调优合同解析」，target_id 用 `contract-parse` 即可。

**新增技能 checklist：**

- [ ] 写 Target 配置（复制 `spec/01-tunable-target-template.yaml`）
- [ ] 准备 benchmark 或依赖 run-log 收集
- [ ] `POST /api/v1/targets` 注册
- [ ] 在 `auto-tune-operator` Skill 映射表加一行

## 六、手动 vs 自动触发（别人怎么控制）

| 谁 | 怎么做 | 场景 |
|----|--------|------|
| 业务用户 | 对总 Agent 说「调到 90%」 | 半自动 |
| 运维 | `curl POST .../tune/sessions` | 手动 |
| 运维 | 平台 API 文档 `/docs` 点运行 | 手动 |
| 定时任务 | cron 调 `tune.start` | 全自动 |
| 调优助手 Agent | 用户自然语言 | 半自动 |

**默认建议：** 业务方用 Agent/工作流引用（半自动），运维用 cron（全自动），晋升版本人工确认。

## 十、常见问题

**Q：同事还要装 Python 吗？**  
A：不用。只配置平台里的 Agent/工作流/MCP 地址。

**Q：能多个业务共用一个 Tune Engine 吗？**  
A：能。每个业务注册不同 `target_id` 即可。

**Q：调优完怎么生效？**  
A：`tune.get_result` 返回 `best_entry_ref`，在平台把生产 Agent/工作流切到该版本（目前建议人工晋升）。

**Q：和单次重试冲突吗？**  
A：不冲突。普通提问走重试；说「调优」走调优助手。
