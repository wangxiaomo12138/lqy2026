# 单次循环（在线重试）他人接入指南

> **是的，单次循环也是通用的，可以像 Tune Engine 一样给别人直接接入。**  
> 不用写代码：部署 MCP → 引用 Agent/工作流 → 挂 Skill，任何业务都能复用。

---

## 一、和 Tune Engine 的对比

| | 单次循环（在线重试） | Tune Engine（批量调优） |
|--|---------------------|-------------------------|
| **通用吗** | ✅ 是 | ✅ 是 |
| **别人怎么接** | 引用 retry-assistant / wf_retry_wrapper | 引用 tune-assistant / wf_tune_generic |
| **依赖服务** | check-mcp :8200 + run-log-mcp :8300 | tune-engine :8100 |
| **何时生效** | 用户**每次提问**自动重试 | **手动/定时**批量调优 |
| **要注册吗** | 配 `task_type` + `expected_fields` 即可 | 要注册 Target + benchmark |

## 二、别人接入（3 选 1）

### 方式 A：引用「在线重试助手 Agent」★ 推荐

**配置文件：** `integrations/agents/retry-assistant.json`

#### 步骤

1. 部署服务（若已部署 Tune Engine 全套，check-mcp / run-log-mcp 已有）：

```bash
bash scripts/start-all.sh
```

2. 打开 `integrations/agents/retry-assistant.json`，`YOUR_HOST` → 实际 IP

3. 平台 **新建 Agent**：`在线重试助手`

4. 配置：

| 配置项 | 值 |
|--------|-----|
| MCP 1 | `http://HOST:8200/sse`（SSE）→ 工具 `check.evaluate` |
| MCP 2 | `http://HOST:8300/mcp` → `run.log` / `run.stats` |
| Skill | 复制 `skills/auto-retry-replan/SKILL.md` |

5. **总 Agent 添加 Agent 引用** → `在线重试助手`

#### 使用效果

```text
用户：解析这份合同，输出 JSON……

总 Agent → 在线重试助手（或带 Skill 的业务流）
  → 执行合同解析工作流
  → check.evaluate（不好？）
  → run.log（记录）
  → 重规划再执行
  → 输出最优结果
```

```text
┌──────────┐  引用   ┌────────────────┐  MCP   ┌────────────┐
│  总 Agent │ ─────> │ 在线重试助手     │ ─────> │ check-mcp  │
└──────────┘        │ + 业务工作流     │        │ run-log-mcp│
                      └────────────────┘        └────────────┘
```

### 方式 B：引用「重试包装工作流」

**配置文件：** `integrations/workflows/wf_retry_wrapper.json`

适合：平台以**工作流**编排为主。

1. 平台新建工作流 `wf_retry_wrapper`，复制 JSON 里 5 步 intent
2. 总 Agent 挂 check-mcp + run-log-mcp + Skill `auto-retry-replan`
3. **业务 query 先走 wf_retry_wrapper**，内部再路由到具体业务工作流（合同解析、发票等）

```text
总 Agent
  → wf_retry_wrapper（通用重试壳）
       → wf_contract_parse（你的业务）
       → wf_invoice_parse（另一个业务）
```

**一个重试工作流，包所有业务。**

---

### 方式 C：直接挂到业务 Agent（最常用）

不新建 Agent，在**现有合同解析 Agent** 上加 3 样东西：

1. MCP check-mcp + run-log-mcp
2. Skill `auto-retry-replan`
3. 工作流描述加一句：「执行后必须 check.evaluate + run.log」

适合：只有某一个业务要重试，不想动总 Agent 结构。

## 三、通用性：新业务怎么接（不用改 MCP 代码）

单次循环靠 **`task_type` + `expected_fields`** 区分业务，在 Skill 或工作流里配置即可。

### 注册新业务（复制改表）

在 `integrations/task-registry.yaml` 或 Skill 里加一行：

```yaml
invoice-parse:
  task_type: invoice-parse
  expected_fields: [seller, buyer, amount, tax_id, invoice_date]
  workflow_ref: wf_invoice_parse
  trigger_keywords: [发票, 开票, 税号]
```

规划模型看到「发票」→ 走 invoice-parse 的 expected_fields 调 check / run.log。

### 调用示例（规划 Step 2 自动执行）

```
check.evaluate:
  task_type: "invoice-parse"
  output: <工作流输出>
  expected_fields: ["seller","buyer","amount","tax_id","invoice_date"]

run.log:
  target_id: "invoice-parse"
  task_type: "invoice-parse"
  query: <用户原始 query>
  output: <工作流输出>
  summary: <总结>
  expected_fields: [...]
  attempt: 1
```

**合同、发票、报表……同一套 MCP，只改配置。**

## 四、与 Tune Engine 联合接入（总 Agent 最终形态）

```text
总 Agent
├── 引用：在线重试助手（或 wf_retry_wrapper）  ← 普通提问，每次自动重试
├── 引用：调优助手（或 wf_tune_generic）        ← 用户说「调到90%」
├── 引用：合同解析工作流 / 发票工作流 …          ← 具体业务
└── MCP：8200 + 8300 + 8100
```

| 用户说什么 | 走什么 |
|------------|--------|
| 「解析这份合同」 | 业务工作流 + 单次循环 |
| 「调到 90%」 | Tune Engine 批量调优 |
| 普通提问 | 单次循环自动生效，用户无感 |

## 五、接入检查清单

**单次循环（给别人用前确认）**

- [ ] check-mcp 已部署（:8200）
- [ ] run-log-mcp 已部署（:8300）
- [ ] 总 Agent 或业务 Agent 已挂两个 MCP
- [ ] Skill `auto-retry-replan` 已绑定
- [ ] 已引用 retry-assistant 或 wf_retry_wrapper（三选一）
- [ ] 业务 `task_type` / `expected_fields` 已配置
- [ ] 测试 query 后规划轨迹含 `check.evaluate` 和 `run.log`

## 六、验证命令

```bash
# 检查服务
curl http://HOST:8200/health
curl http://HOST:8300/health

# 模拟一次检查+记录
curl -X POST http://HOST:8200/mcp/tools/call \
  -H "Content-Type: application/json" \
  -d '{"tool":"check.evaluate","arguments":{"task_type":"contract-parse","output":{"party_a":"甲"},"expected_fields":["party_a","party_b","amount"]}}'

curl -X POST http://HOST:8300/mcp/tools/call \
  -H "Content-Type: application/json" \
  -d '{"tool":"run.log","arguments":{"target_id":"contract-parse","query":"解析合同","output":{"party_a":"甲"},"summary":"缺字段","expected_fields":["party_a","party_b","amount"]}}'
```
